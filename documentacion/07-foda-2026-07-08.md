# FODA de InsightPulse

Fecha: 8 de julio de 2026.

## Fortalezas

- Ya existe una base SaaS real, no solo una demo local.
- El producto esta desplegado en Cloudflare con frontend, API, base de datos y gateway de IA.
- La calidad del analisis mejoro mucho al pasar de TextBlob a Gemini.
- La separacion entre demo y producto real cuida la base de datos y los costos.
- Ya hay autenticacion inicial con usuario, contrasena, sesion y workspace.
- El panel privado tiene modulos claros: overview, uso, CSV, analisis e historial.
- El historial permite eliminar feedback, lo que ayuda a evitar acumulacion de basura.
- Las metricas de uso ya registran fallos, 429, tokens y latencia.
- El dashboard ya empieza a mostrar lectura ejecutiva, areas, temas y prioridades.
- El reporte exportable convierte analisis guardados en material util para decisiones.
- El backend reutiliza duplicados exactos para evitar gasto innecesario de LLM.
- El proyecto tiene una direccion comercial clara: plan unipersonal para emprendedores.
- La arquitectura cloud elegida es liviana, barata y escalable para MVP.

## Oportunidades

- Convertir la carga CSV en una experiencia de analisis por lote con resumen ejecutivo.
- Agregar soporte para Excel y PDF para cumplir mejor la promesa del plan unipersonal.
- Mejorar los reportes exportables hasta convertirlos en entregables para reuniones o decisiones.
- Sumar billing cuando el valor del flujo este validado.
- Usar las metricas de uso para disenar limites de plan y cuidar margen.
- Incorporar integraciones simples mas adelante, como Google Sheets o formularios.
- Desarrollar dashboards por problema, oportunidad, area y riesgo.
- Posicionar el producto como inteligencia de feedback para negocios chicos que no tienen equipo de data.
- Aprovechar Cloudflare para seguir iterando rapido con bajo costo inicial.

## Debilidades

- Todavia no hay registro de usuarios desde la app.
- No hay billing real ni control comercial de suscripciones.
- El soporte de archivos esta limitado a CSV; Excel y PDF todavia no estan implementados.
- El dashboard ya tiene una primera lectura de negocio, pero todavia necesita mas profundidad por patrones y oportunidades.
- Falta una suite de tests mas completa para asegurar cambios futuros.
- La autenticacion inicial funciona, pero requiere endurecimiento antes de vender a clientes reales.
- La cuota gratuita de Gemini limita las pruebas intensivas.
- Todavia no hay filtros avanzados en historial.
- No hay permisos por roles ni gestion multiusuario.
- El producto aun necesita una narrativa comercial mas compacta y casos de uso mas demostrables.

## Amenazas

- La cuota o el costo del proveedor LLM puede afectar el margen si no controlamos limites por plan.
- Productos grandes de CRM, soporte o analytics pueden sumar funciones similares.
- Un analisis incorrecto podria generar perdida de confianza si no mostramos contexto y limites.
- Manejar feedback real implica expectativas de privacidad, seguridad y retencion de datos.
- Si el onboarding no es muy simple, el usuario podria abandonar antes de ver valor.
- Si agregamos demasiadas funciones antes de validar el flujo central, el producto puede volverse dificil de mantener.
- Dependencia fuerte de Gemini: cambios de precio, cuota o disponibilidad impactan el servicio.
- La falta de billing y alta automatica impide escalar ventas sin intervencion manual.

## Lectura final

InsightPulse esta bien posicionado para seguir como MVP SaaS. La base tecnica ya permite demostrar valor real, pero el producto todavia necesita cerrar una promesa muy concreta.

La mejor siguiente jugada es fortalecer el plan unipersonal:

```text
Carga un archivo simple y recibe un resumen ejecutivo con problemas, oportunidades, patrones y acciones sugeridas.
```

Eso convierte la herramienta en algo que una persona puede entender, probar y eventualmente pagar.
